# AR-141 — Timestamp-safe index addition/deletion announcement drift in liquid U.S. large-cap common stocks

## Alpha Hypothesis
- Claim: Actual public index addition/deletion announcement records, used only after a conservative announcement-time lag, may identify temporary flow/attention pressure and underreaction in liquid U.S. large-cap common stocks. Additions and deletions should be tested separately for post-announcement drift and later reversal.
- Economic / behavioral intuition: Index membership changes can create benchmark-flow, arbitrage, liquidity, and attention effects. Unlike rejected deterministic calendar/rebalance proxies, this uses true public addition/deletion announcements with timestamp discipline.
- Expected failure mode: Historical announcement sources may be incomplete, revised, paywalled, or not timestamp-safe; results may reduce to generic momentum/reversal, June/Russell seasonality, size/liquidity, sector beta, or one index-family/event-year cluster.

## Implementation Plan
- Candidate product pool: All U.S. common-stock additions/deletions for S&P 500, Nasdaq-100, Russell 1000/large-cap or equivalent large-cap Russell events with reproducible public announcement records and Alpaca/qfa real daily bars.
- Product-selection filters: timestamp-safe announcement date/time or conservative next-session lag; common-stock-like symbols only; exclude ETFs/funds/preferreds/warrants/OTC where identifiable; minimum price, liquidity, and qfa/Alpaca coverage; no return-based selection.
- Selected universe: To be selected before scoring. Block/reject if fewer than 150 timestamp-safe, tradable, liquid common-stock events survive.
- Universe-selection rationale / limitations: Broad large-cap index-membership event pool attempts to avoid sparse single-index annual calendar tests. Deletion events may suffer survivorship and current-symbol mapping limitations; record dropped/ambiguous events explicitly.
- Data source: Public index addition/deletion announcement records plus Alpaca/qfa real daily OHLCV. No CSV-backed market data, no qfa daemon, no orders.
- Signal features: index family, add/delete flag, announcement-to-effective gap, next-session return/reaction diagnostics, liquidity/size filters, event timing.
- Model form: Timestamp-safe event study / allocator with additions and deletions evaluated separately; no ex-post blending.
- Parameters / search space: next-session entry after conservative lag; fixed 1d/5d/10d/20d hold diagnostics with predeclared primary; 10 bps one-way cost primary, 5/20 bps sensitivity.
- Position sizing: equal-risk or equal-weight active event baskets with symbol/index-family/year caps; cash when no events.
- Risk constraints: no same-day lookahead; separate announcement vs effective date; leave-year and index-family concentration diagnostics; do not silently drop hard-to-map deletions.

## Evaluation Plan
- Random-period protocol: Feasibility/event-source gate first; if passed, evaluate random event-containing windows and leave-year-out diagnostics over the timestamp-safe source/qfa overlap.
- Metrics: median/p25/worst random-window Sharpe, positive-window rate, annualized return/volatility, max drawdown, turnover, activation/event count, event concentration by index family/year/add-delete side.
- Costs/slippage assumptions: primary 10 bps one-way; 5/20 bps sensitivity.
- Orthogonality checks vs alpha library: hard controls against AR-106 OPEX/rebalance liquidity-pressure rejection, AR-126 post-OPEX single-name calendar-flow rejection, Russell reconstitution size-spread rejection precedent, AR-130 public filing-calendar attention rejection, generic momentum/reversal, sector/beta residuals, shifted announcement dates, same-size/liquidity non-event matched names, same-month/year random events, June/rebalance seasonality, and OPEX/rebalance overlap.
- Acceptance threshold: event-source gate passes; positive median after-cost Sharpe; p25 not materially negative; positive-window rate >=55%; 20 bps does not collapse; primary beats matched non-events and generic momentum/reversal/calendar controls; no single index family/year/rebalance season dominates; max relevant correlation <=0.60.
