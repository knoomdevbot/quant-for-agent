## Alpha Hypothesis
- Claim: Liquid U.S. equities may exhibit short-horizon drift or reversal after stock split effective/ex/process dates because split events change retail attention, displayed price, liquidity, and post-event participation. The evaluation must trade only after the effective/ex/process date and must not rely on unavailable announcement timestamps.
- Economic / behavioral intuition: Forward splits can attract attention and liquidity after the split-adjusted price appears lower, while reverse splits may reflect distress and should be separated or excluded from the primary sleeve.
- Expected failure mode: Event count is sparse, dominated by a few mega-cap tech split episodes, explained by simple momentum/reversal/beta, or blocked because Alpaca split corporate-action records are unavailable/ambiguous.

## Implementation Plan
- Candidate product pool: Broad liquid U.S. common stocks available through Alpaca real daily OHLCV, intersected with timestamp-safe Alpaca corporate-action split records (forward/reverse/unit split types where available). Start from a broad ex-ante split-event pool; do not select by realized return.
- Product-selection filters: Sufficient pre/post daily bars, high dollar volume before the event, event-date fields present and non-conflicting, no penny/distressed names in the primary forward-split sleeve, minimum event dispersion across symbols and years.
- Selected universe: To be finalized from split-event symbols after data-availability and liquidity filters. Primary sleeve should prefer forward splits; reverse/unit splits must be separate diagnostics unless enough clean liquid events justify inclusion.
- Universe-selection rationale / limitations: The idea needs event-specific single-name products rather than a repeated ETF basket. Main limitations are sparse split events, adjusted-price artifacts, corporate-action field semantics, and concentration in a few high-attention mega-cap names.
- Data source: Alpaca real daily OHLCV plus Alpaca corporate-action split records accessed through alpaca-py if available. No CSV, no daemon, no orders.
- Signal features: Post-event 1/3/5/10 day drift and reversal legs after the event date; liquidity/volume filters only from pre-event data; market/beta residualization diagnostics.
- Model form: Rule-based event study with fixed post-effective-date entry/exit windows.
- Parameters / search space: Holding windows {1,3,5,10} trading days; split type {forward primary, reverse/unit diagnostics}; costs {5,10,20 bps}; liquidity thresholds; optional beta-residualized return ranking only as diagnostic.
- Position sizing: Equal-risk/equal-notional event sleeves capped per symbol/event; long-only or market-neutral diagnostics must be explicitly separated.
- Risk constraints: No pre-effective-date exposure; max concentration cap by symbol/year; event-count minimum; reject if adjusted-price mechanics or one event/year dominates.

## Evaluation Plan
- Random-period protocol: Use random event/time windows across eligible history; compute event-year and symbol leave-one-out diagnostics. Trade from the next session after event date only. Include same-symbol non-event placebo dates and randomized split dates within the same month/year.
- Metrics: Median Sharpe, p25 Sharpe, worst-window Sharpe, annualized/event return, volatility, max drawdown, turnover, hit rate, positive-window rate, capacity proxy via dollar volume and event count.
- Costs/slippage assumptions: 10 bps primary, with 5 bps and 20 bps sensitivity.
- Orthogonality checks vs alpha library: Correlate compact return stream against watchlist liquidity-reversal, event-drift, momentum/reversal, and single-name models where comparable; mark deferred_due_rejection if rejected before orthogonality can matter.
- Acceptance threshold: 10 bps median random-window Sharpe > 0, p25 not materially negative, positive-window rate >= 55%, beats simple momentum/reversal/beta/placebo baselines, not dominated by one symbol/year, and max relevant correlation <= 0.60 hard / <= 0.50 preferred.
