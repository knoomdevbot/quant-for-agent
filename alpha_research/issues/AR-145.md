## Alpha Hypothesis
- Claim: Timestamped SEC 8-K item-specific filings, especially Item 2.02 earnings-results filings, may identify post-filing drift or reversal in liquid U.S. common stocks when traded only after a conservative acceptance-time lag.
- Economic / behavioral intuition: Unlike rejected deterministic filing-calendar effects, this uses actual event content/item tags and EDGAR accepted timestamps. Item-specific corporate updates can concentrate new information and attention; delayed digestion may create short-horizon drift or overreaction.
- Expected failure mode: The signal reduces to generic post-earnings drift, momentum/reversal, market beta, sector effects, issuer/year concentration, or filing-time artifacts. Current CIK-to-ticker mapping may introduce survivorship/ticker-change bias. Item tags or document contents may be too noisy for a reproducible event table.
- Falsifier: If a timestamp-safe, reproducible 8-K item/ticker event table cannot be built with sufficient liquid common-stock breadth, mark blocked/feasibility-only and do not run a performance backtest. If performance testing later fails after costs versus matched non-events, shifted labels, momentum/reversal, and AR-130-style filing-calendar controls, reject.

## Implementation Plan
- Candidate product pool: All U.S. exchange-listed common-stock issuers with SEC submissions API 8-K filings over the qfa/Alpaca overlap, starting with filings whose `items` field includes Item 2.02. Items 7.01 and 8.01 may be diagnostic sleeves only after feasibility is proven.
- Product-selection filters: SEC `form == 8-K`; non-missing `acceptanceDateTime`, accession number, item tags, and primary document handle; CIK-to-ticker mapping; qfa/Alpaca daily bar coverage; common-stock-like instruments only; exclude ETFs/funds/ADRs/preferreds/warrants where identifiable; no return-based selection.
- Selected universe: To be fixed before scoring. Feasibility gate should require a broad liquid mapped event set across many issuers and years; target at least 300 Item 2.02 liquid common-stock events and no single issuer/year/month dominance before performance evaluation.
- Universe-selection rationale / limitations: SEC accepted timestamps provide a point-in-time event anchor; current SEC ticker mapping may miss historical ticker changes and delisted names, so survivorship/ticker-change limitations must be explicitly recorded.
- Data source: SEC submissions/company-tickers public endpoints for event metadata; Alpaca/qfa real daily OHLCV for market bars if and only if feasibility gates pass. No CSV-backed market data, no qfa daemon, no orders.
- Signal features: form type, item tag, accepted timestamp, next-session lag, event side fixed as drift/reversal diagnostics, issuer/event concentration, optional after-hours vs regular-hours indicator. Do not parse sentiment unless a reproducible timestamp-safe parser is implemented.
- Model form: Feasibility-first event-study scout; later rule-based event drift/reversal model only after event-table gates pass.
- Parameters / search space: Primary Item 2.02 sleeve; 1d/5d/10d/20d hold diagnostics; 10 bps one-way primary cost; 5/20 bps sensitivity; conservative next-session entry for all accepted times.
- Position sizing: Equal-weight active event basket with issuer, sector, and event-date caps; cash when no events.
- Risk constraints: No same-day lookahead; no current-only ticker mapping without caveat; drop ambiguous mappings; cap clustered mega-cap earnings days; record dropped events.
