## Alpha Hypothesis
- **Claim:** Option-implied volatility and rate-sensitive ETF realized-volatility shocks around scheduled macro events may identify event-risk repricing that can be allocated across duration, gold, credit, and equity ETF sleeves.
- **Economic / behavioral intuition:** Unlike the rejected AR-091 prediction-market path, listed ETF/asset-market implied or realized risk proxies are directly tradable-market signals and can be selected ex ante from scheduled event windows without relying on a point-in-time prediction-market universe archive.
- **Expected failure mode:** Event windows are too sparse, realized-vol proxies lag true option-implied risk, or the signal is just another duration/equity momentum variant.
- **Falsifier:** Median random-window Sharpe <= 0 after costs, materially negative p25 Sharpe, instability around non-Fed macro regimes, or high redundancy with existing ETF defensive/momentum alphas.

## Implementation Plan
- **Candidate product pool:** Liquid event-sensitive ETFs selected ex ante: duration curve ETFs, credit ETFs, gold/commodities, broad equity indexes, volatility proxies if qfa data-supported, and defensive sectors.
- **Product-selection filters:** Alpaca/qfa data availability, liquidity, economic exposure to rate/event risk, history through multiple Fed/inflation regimes, and no ex-post return selection.
- **Selected universe:** Target 20-50 liquid ETFs after filtering; start from TLT, IEF, SHY, HYG, LQD, GLD, SPY, QQQ, IWM, XLU, XLP, XLV, XLE, XLF, plus VIX/volatility proxies only if real data is available.
- **Universe-selection rationale / limitations:** Divergent from AR-091 because it uses tradable-market volatility/event-risk proxies rather than prediction-market probabilities; may be limited if true listed option IV data is unavailable and must be proxied by realized range/volume.
- **Data source:** Alpaca real OHLCV via qfa; optional real option-implied data only if a reproducible provider is already available. No CSV-backed research.
- **Signal features:** Pre-event realized range/volume z-scores, duration/equity/gold stress breadth, event-window gating, volatility compression/expansion state.
- **Model form:** Rule-based qfa `generate_signals(context)` allocator.
- **Parameters / search space:** lookback 10/20/60, event window +/- 1/3/5 trading days, range z-threshold 1.0/1.5/2.0, max sleeve weight 0.35.
- **Position sizing:** Gross <= 1.0, no leverage, SHY/cash-like fallback when event-risk confidence is low.
- **Risk constraints:** No daemon, no orders, no secrets, no raw paid data committed.

## Evaluation Plan
- **Random-period protocol:** qfa/Alpaca smoke test, then 8-12 random/stress windows including scheduled macro-event-heavy and quiet periods.
- **Metrics:** Sharpe distribution, annualized return/volatility, max drawdown, turnover, hit rate, activation rate.
- **Costs/slippage assumptions:** 5-10 bps one-way turnover cost proxy.
- **Orthogonality checks vs alpha library:** Compare to ETF TSMOM, defensive carry, stress/carry, and liquidity reversal artifacts where retained return streams exist; otherwise report unavailable.
- **Acceptance threshold:** Positive median and non-disastrous p25 Sharpe after costs, acceptable drawdown/turnover, and low redundancy.
