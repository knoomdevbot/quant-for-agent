## Alpha Hypothesis
- Claim: Actual EDGAR acceptance-timestamped 8-K filings for specific high-information item buckets may create short-horizon post-filing drift or reversal in liquid U.S. common stocks, distinct from deterministic filing-calendar effects.
- Economic / behavioral intuition: Certain unscheduled corporate disclosures may be underreacted to or overreacted to, especially after-hours or in broad/noisy item buckets, while deterministic filing-deadline calendars are already known and have failed locally.
- Expected failure mode: 8-Ks are widely monitored; item categories are broad/noisy; effects collapse to generic event drift, earnings contamination, momentum/reversal, liquidity/size/sector beta, or a few mega-cap/year clusters.

## Implementation Plan
- Candidate product pool: Liquid U.S. common-stock-like issuers with SEC submissions API records exposing CIK, ticker mapping, 8-K `acceptanceDateTime`, `items`, accession number, primary document, and qfa/Alpaca real daily bars.
- Product-selection filters: common stocks only where identifiable; exclude ETFs/funds/ADRs/preferreds/warrants/OTC when identifiable; minimum price/liquidity/coverage; conservative CIK/ticker mapping; drop ambiguous mappings; no return-based selection.
- Selected universe: To be selected after event-source gate. Minimum event-source gates: >=150 timestamp-safe liquid common-stock events overall, >=50 per primary item bucket or predeclared pooled high-information bucket, >=4 calendar years, and no excessive concentration by one year/item/name/sector.
- Universe-selection rationale / limitations: SEC submissions directly expose item labels and acceptance timestamps, materially different from rejected deterministic SEC 13F deadline AR-130 and easier than blocked Form 4 parsing AR-134. Current-ticker survivorship and item noise must be labeled.
- Data source: SEC submissions API / company_tickers mapping for event timestamps and items; qfa/Alpaca real daily OHLCV for returns. No CSV-backed market data, no qfa daemon, no orders.
- Signal features: 8-K item bucket, acceptance-time bucket (pre-open, regular session, after-hours, weekend), optional pre-filing reaction/volume diagnostics, issuer liquidity/sector controls.
- Model form: Feasibility-first timestamped event-study allocator; separate drift/reversal sleeves and item buckets; no ex-post blending.
- Parameters / search space: Exclude pure earnings Item 2.02 from primary; primary buckets 5.02 leadership changes and 8.01 material events, with 2.05 restructuring/costs diagnostic or pooled if sparse; holds 1/3/5/10 trading days; entry no earlier than next tradable daily bar after EDGAR acceptance.
- Position sizing: Equal-weight or equal-risk active event baskets with issuer/year/item/sector caps; cash when no events.
- Risk constraints: conservative timestamp-to-trade discipline; no same-close lookahead; no uncontrolled HTML/NLP crawl; compact event summaries only.

## Evaluation Plan
- Random-period protocol: After source gate, evaluate random event-containing windows plus leave-year-out diagnostics across the SEC/qfa overlap.
- Metrics: median/p25/worst random-window Sharpe, positive-window rate, annualized return/volatility, max drawdown, turnover, event count, hit rate, item/year/name/sector concentration, activation rate.
- Costs/slippage assumptions: primary 10 bps one-way; 5/20 bps sensitivity.
- Orthogonality checks vs alpha library: Compare against AR-130 public filing-calendar rejection, AR-134 Form 4 blocked event-source precedent, AR-014/024/030/096 event-drift/earnings proxy failures, generic all-8-K drift, shifted labels (+/-5/+/-20 trading days), same-symbol random dates matched by weekday/month/year, matched non-event symbols, generic momentum/reversal, and filing-calendar-only controls.
- Acceptance threshold: Source gate passes; positive after-cost median random-window Sharpe; p25 not materially negative; positive-window rate >=55%; beats shifted labels, generic event drift, generic momentum/reversal, filing-calendar-only and matched non-event controls; no single item/year/name/sector dominates; max relevant correlation <=0.60 where streams are available.

## Preflight Rationale
Independent preflight returned `queue` with medium-low confidence (23/30) as a feasibility-first scout. SEC public endpoints were reachable during preflight: `company_tickers.json` and submissions API expose `acceptanceDateTime`, `form`, `items`, `accessionNumber`, and `primaryDocument`; a bounded 49-symbol liquid sample found all symbols mapped and 1,517 target-item hits since 2020 (5.02: 655, 8.01: 851, 2.05: 11). This is materially distinct from deterministic 13F deadline AR-130 and simpler than Form 4 XML/source construction AR-134, but local priors for event/calendar/public filing drift are hostile, so implementation must enforce strict gates, controls, and no ex-post blending.
