## Alpha Hypothesis
- Claim: Public SEC N-PORT/NPORT-P filings, traded only after EDGAR acceptance/public availability, may reveal fund/ETF holdings changes that forecast short-horizon ownership/flow pressure or reversal in liquid affected common stocks, sectors, or ETFs.
- Economic / behavioral intuition: Large disclosed additions/reductions by funds can proxy for ownership pressure, crowding, or delayed copycat/flow effects, but only after the information is public. The mechanism uses actual portfolio contents rather than deterministic filing calendars or price-only pressure proxies.
- Expected failure mode: Holdings are stale by the time they are public; parser/mapping breadth collapses; amendments or CUSIP/ticker changes introduce lookahead/survivorship; signal is just generic momentum/reversal, sector momentum, 13F filing-calendar attention, or fund-family concentration.
- Falsifier: If a timestamp-safe event table cannot be built from EDGAR acceptance/public timestamps, report period dates, holdings identifiers, and tradable security mappings with sufficient breadth, stop at source gate. If later performance fails shifted-publication-date, shuffled-holdings, momentum/reversal, sector, 13F-calendar, and concentration controls, reject.

## Implementation Plan
- Candidate product pool: SEC EDGAR N-PORT/NPORT-P structured filings for registered funds/ETFs; liquid U.S. common stocks, sector/industry ETFs, and fund exposure buckets that can be mapped from disclosed holdings after public availability.
- Product-selection filters: EDGAR acceptance/public timestamp availability; report period and amendment handling; extractable holding identifiers, shares, market value, issuer names; PIT-safe or conservative CUSIP/ticker/security mapping; liquidity/data availability through qfa/Alpaca real daily bars; breadth and concentration gates.
- Selected universe: To be fixed only after the source gate. Minimum gate: at least 150 liquid issuer/exposure events after timestamp, mapping, liquidity, and concentration filters; otherwise source-gate reject/block before performance testing.
- Universe-selection rationale / limitations: The mechanism needs broad fund-disclosure events and tradable mapped securities; a narrow repeated ETF basket would not test the N-PORT holdings-change mechanism.
- Data source: SEC EDGAR N-PORT/NPORT-P filings and acceptance/public timestamps for signal; Alpaca/qfa real daily OHLCV for returns only if source gate passes. No CSV-backed qfa market data, no daemon, no orders.
- Signal features: Fund-level and aggregate holdings additions/reductions, dollar/share changes, issuer/security exposure deltas, possible fund-family concentration and cross-fund breadth; all computed from filings public before the trading decision.
- Model form: Source-gate feasibility scout first; if feasible, event-driven/ranked rule-based allocator over mapped liquid securities or exposure ETFs.
- Parameters / search space: Conservative public-availability lag; event windows 1/5/10/20 trading days; holdings-change z-score/percentile thresholds; breadth and concentration filters; 5/10/20 bps cost sensitivity.
- Position sizing: Gross exposure <=1.0; capped single-name/sector/fund-family exposure; cash on missing/ambiguous mapping; no leverage.
- Risk constraints: No period-end-date lookahead; no trading before EDGAR/public availability; drop ambiguous mappings/amendments where direction cannot be trusted; reject if one fund family, sector, issuer, or year dominates.

## Evaluation Plan
- Random-period protocol: Source/parser/mapping/breadth gate first. If passed, run random event windows and leave-year/fund-family/sector diagnostics over the qfa/Alpaca overlap.
- Metrics: Source coverage, parser success rate, mapping coverage, event count, median/p25/worst random-window Sharpe, positive-window rate, annualized return/volatility if meaningful, max drawdown, turnover, hit rate, activation rate, cost sensitivity, concentration diagnostics.
- Costs/slippage assumptions: Primary 10 bps one-way; 5/20 bps sensitivity; higher stress if single-name turnover is high.
- Orthogonality checks vs alpha library: Compare against AR-130 13F filing-calendar attention, AR-134 Form 4 event-source blocker, AR-145/146 SEC filing source gates, generic momentum/reversal, sector momentum, shifted/publication-date placebos, shuffled holdings changes within fund/month, and fund-family concentration.
- Acceptance threshold: Source gate passes with >=150 liquid mapped events, no timestamp/leakage defect, post-cost median random-window Sharpe > 0, p25 not materially negative, positive-window rate >=55%, primary beats shifted/shuffled/calendar/momentum/reversal controls, and max relevant correlation <=0.60 hard.
